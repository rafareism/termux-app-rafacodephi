# BETA_BOOTSTRAP_STATUS

## Current Status: BOOTSTRAP_DEV_LOCAL_ONLY

The current developer bootstrap generated by `scripts/bootstrap_zip_builder.c` is explicitly marked `BUILD_ONLY` and does **not** contain runtime-complete payload (`SYMLINKS.txt`, `bin/busybox`, `bin/proot`).

## Status Definitions

- `BOOTSTRAP_BUILD_READY`: Build chain resolves and embeds bootstrap artifacts, but runtime guarantees are incomplete.
- `BOOTSTRAP_RUNTIME_READY`: Eligible only when all runtime criteria below are satisfied.
- `BOOTSTRAP_DEV_LOCAL_ONLY`: Developer-local bootstrap flow works for build/test only; must not be treated as production runtime bootstrap.
- `BOOTSTRAP_BLOCKED`: Build or runtime invariants fail and beta must be blocked.

## Runtime-Ready Criteria Gate

Beta can be marked `BOOTSTRAP_RUNTIME_READY` only if all checks pass:

- BLAKE3 configured (`TERMUX_BOOTSTRAP_BLAKE3_AARCH64/ARM/I686/X86_64`).
- Zip includes `BOOTSTRAP_INFO`.
- `TERMUX_PACKAGE_NAME` matches expected package.
- `TERMUX_PAGE_SIZE` matches expected page size.
- `SYMLINKS.txt` present.
- `bin/sh` present.
- `bin/pkg` present.
- `bin/busybox` present.
- `bin/proot` present.
- `BootstrapBaremetalGuard` never crashes with `UnsatisfiedLinkError`.

## Enforcement in this repository

- Release builds fail early when BLAKE3 env vars are missing.
- Developer bootstrap outputs include `BUILD_ONLY` marker.
- Guard native calls are protected against missing JNI symbols in beta mode.
